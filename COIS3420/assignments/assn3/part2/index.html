highlight.js - take your coding to the next level

Introduction

highlight.js is a highly customizable Javascript library that gives you full power over the code highlighting on your page. Get the most popular highlighting styles or create your own with the help of an extensive library API. By leveraging the power of highlight.js, find the best way to present your code!

Discussion on pros and cons

In my opinion, customization of highlight.js comes at the cost of simplicity: getting started is a full process, with various ways to get started with the library. The documentation can also be overwhelming, with several points of information (like GitHub, Library API, and a dedicated website) confusing the potential user. Additionally, developers don't say anything about accessibility, which might create concerns for users who use screen readers. However, the performance is steadfast (with the load times under 1-2 seconds from the Loki server at Trent). Finally, with the heavy customization options, one can expect to increase the performance even further by choosing only the languages that will be present on the page.

Get started

Developers give a variety of options for installing the library. I personally recommend the easiest one: by linking it to the page with one of the styles in the <head> tag: 

                        
<link rel="stylesheet" 
    href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.18.1/styles/default.min.css">
<script src="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.18.1/highlight.min.js"></script>
Right after that, you can include the following line to automatically highlight all code pieces: 
                        
<script>hljs.initHighlightingOnLoad();</script>
This code will run highlighting on all code blocks after the DOM has loaded. To designate a particular part as a code snippet, use <pre><code> tags like this: 
                        
<pre>
    <code>
        <!-- Your code goes here -->
    </code>
</pre>
You are all set!

More features

Specifying a particular language:

By default, highlight.js tries to automatically detect the language. However, what if you want to lock the language? Add a class="desired-language" to <code> element (note: not <pre>!), for example class="java" to force the language highlighting to Java. You can also use 'lang-' and 'language-' prefixes, like this: class="lang-java".

Creating a custom package:

By following this link, you can create a custom package with the languages you will use on your page. Select needed languages, upload the library directly to your server, and enjoy a significant boost in performance!

Using different themes:

By changing the CSS link, you can change the syntax highlighting to a different style you prefer better. For a GitHub style, you can use this link for CSS: 

                        
<link rel="stylesheet"
    href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.18.1/styles/github.min.css">
Click on a button to change style of the entire page to GitHub theme!

Highlighting code blocks separately:

What if you want to highlight the blocks separately? There is a special function for that. By using hljs.highlightBlock(Element element) function, you can turn on highlighting for only specific blocks or activate them when needed. For example:

HTML
                        
<pre class="hglt-after-press">
    <code class="java">
        System.out.println("I love highlight.js!");
    </code>
</pre>    

<button id="hglt-on-press">Highlight!</button>
JS
                        
// Find the code enclosement by selector (you need to select the code, not pre element)
var code_snippet = document.querySelector(".hglt-after-press code");
// Find the button and attach the eventListener that will highlight 
// the code_snippet on a click of a button
var btn = document.querySelector("#hglt-on-press");
btn.addEventListener('click', () => {
    hljs.highlightBlock(code_snippet);
});
To test this code, click the button below! 
                        
System.out.println("I love highlight.js!");

Displaying code as plaintext:

What if you want no highlighting? Just include a class="plaintext" for the <code> element. 

                        
This text is not formatted, as class "plaintext" is used.

Disabling highlighting

What if the highlighting must be completely disabled? You can do that by adding class="nohighlight" to the <code> element. But if you want to disable highlighting after the page was loaded, you can remove the "hljs" from classList of particular code section. Click the button to test this out: 

                        
System.out.println("This library is lit! (Just as my deadlines (I need help please)!)");
